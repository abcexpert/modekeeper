# Внутренний документ: аккорды

Аккорд — это согласованная подстройка 2–3 ручек как единое действие.

## Когда использовать
- Сигнал drift + рост латентности.
- Рост потерь при стабильной загрузке.
- Разбалансировка по узлам (stragglers).

## Примеры аккордов
- `stability_boost`: уменьшить learning rate + усилить регуляризацию.
- `throughput_rebalance`: увеличить batch size + снизить concurrency.
- `straggler_relief`: уменьшить нагрузку на отстающих + увеличить тайм-ауты.

## Почему аккорды
- Изменения координированы и уменьшают риск одиночных перекосов.
- Позволяют фиксировать комплексные стратегии в policy.
- Упрощают explain-log и аудит.
